# Autonomous Disaster Management System

Welcome to the **Autonomous Disaster Management System**! This project leverages the power of Large Language Models (LLMs), AI Agents, and Machine Learning to create a comprehensive pipeline for analyzing environmental conditions, predicting potential disasters, and generating actionable alerts. 

The system incorporates a **Human-in-the-loop** workflow for alert review and a **Self-Improving Memory** module to learn from human feedback and refine its future recommendations.

## 🌟 Key Features

* **AI Agent Pipeline (LangGraph & LangChain):** A structured workflow of specialized nodes (Agents) handling weather ingestion, forecasting, disaster prediction, news monitoring, cognitive assessment, and departmental action planning.
* **Human-in-the-Loop Review:** Alerts generated by the system are held for human review. Gatekeepers can either approve and dispatch alerts or reject them with feedback.
* **Self-Improving Memory:** When an alert is rejected, the system uses the provided feedback to generate "Learned Insights." These insights are fed back into the pipeline, allowing the agents to self-improve and produce better alerts in subsequent iterations.
* **Predictive Modeling:** Integrates machine learning (Scikit-learn, Pandas) for time-series forecasting and disaster prediction.
* **Interactive Web Interface:** A sleek dashboard served via Flask for interacting with the pipeline, reviewing alerts, and viewing learned insights.

## 🏗️ Architecture

The system's intelligence is powered by a graph of distinct nodes:
1. **Weather Ingestion Node:** Gathers real-time environmental data.
2. **Time Series Forecast Node:** Predicts future trends based on historical data.
3. **Disaster Prediction Node:** Evaluates the risk of specific disasters.
4. **News Monitor Node:** Scans for relevant contextual information.
5. **Cognitive Assessor Node:** Synthesizes data to assess overall severity.
6. **Department Agent Node:** Generates department-specific alerts and action plans.
7. **Human Decision Gate:** Pauses the pipeline for a human to review the alert.
8. **Self-Improver Node:** If rejected, analyzes feedback to generate new rules and insights for future runs.

## 🛠️ Technology Stack

* **Backend Framework:** Flask, Flask-CORS
* **AI & Orchestration:** LangChain, LangGraph, Google GenAI
* **Data Science & ML:** Scikit-learn, Pandas, NumPy, Statsmodels
* **Frontend:** Static HTML/JS/CSS (served by Flask)

## 🚀 Getting Started

### Prerequisites

Ensure you have Python 3.10+ installed.

### Installation

1. **Clone the repository:**
   ```bash
   git clone <your-repository-url>
   cd Disaster-Management
   ```

2. **Create a virtual environment (optional but recommended):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use: venv\Scripts\activate
   ```

3. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Environment Variables:**
   * Copy the `.env.example` file to `.env` and fill in your API keys (e.g., Google Gemini API key, weather API keys if applicable).
   ```bash
   cp .env.example .env
   ```

### Running the Application

1. **Start the Flask server:**
   ```bash
   python app.py
   ```

2. **Access the Web Dashboard:**
   Open your browser and navigate to `http://localhost:5000`

## 🧠 How the Self-Improving Loop Works

1. **Run Pipeline:** The user requests an analysis for a specific location. The agents process the data and generate an alert.
2. **Review:** The alert is presented on the dashboard. The user can either **Approve & Send Alert** or **Reject & Improve**.
3. **Provide Feedback:** If rejecting, the user provides specific feedback (e.g., "The alert is too alarming for normal weather patterns").
4. **Learn & Re-run:** The Self-Improver agent processes this feedback, generates a new permanent insight rule, and automatically triggers the Department Agent to rewrite the alert applying the new rule.
5. **Continuous Improvement:** These insights are saved and applied to all future alerts, making the system smarter over time.
